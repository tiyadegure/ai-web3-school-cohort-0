## 二、Proposal Summarizer 草图

### 2.3 AI 做什么 vs 人做什么

## 三、Meeting-to-Action Workflow 草图

### 3.1 工作流程

```
会议结束（Zoom/Google Meet）
         │
         ▼
┌─────────────────────────┐
│  1. 转录 & 分段         │  ← Whisper/API 转录
│  - 按发言人分段         │
│  - 按话题分段           │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  2. AI 提取 Action Items│  ← LLM 结构化提取
│  - 谁负责 (Who)         │
│  - 做什么 (What)        │
│  - 截止时间 (When)      │
│  - 上下文链接 (Context) │
│  - 决策状态 (Status)    │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  3. 决策状态分类        │
│  - 已决定 ✅            │
│  - 待确认 ⏳            │
│  - 需要投票 🗳️          │
│  - 需要预算 💰          │
│  - 需要法律/安全复核 ⚖️ │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  4. 分配 & 追踪         │  ← 任务管理工具
│  - 创建 GitHub Issue    │
│  - 分配到人             │
│  - 设置截止日期         │
│  - 自动提醒             │
└────────┬────────────────┘
         │
         ▼
┌─────────────────────────┐
│  5. 完成验证            │
│  - 负责人提交交付物     │
│  - AI 验证完整性        │
│  - 关闭 Issue           │
└─────────────────────────┘
```

## 五、三个草图的协同关系

```
Proposal Summarizer          Meeting-to-Action          Budget Checklist
    (信息层)                    (执行层)                    (资金层)
        │                          │                          │
        ▼                          ▼                          ▼
  提案摘要 → 讨论    会议决策 → Action Items    拨款 → 里程碑 → 付款
        │                          │                          │
        └──────────┬───────────────┘                          │
                   ▼                                          │
            贡献图谱 ←─────────────────────────────────────────┘
                   │
                   ▼
            声誉系统（ERC-8004）
```

**协同逻辑：**
- Proposal Summarizer 降低信息不对称，帮助社区理解提案
- Meeting-to-Action 把讨论转化为可追踪的执行事项
- Budget Checklist 确保资金使用可审计
- 三者共同输入到贡献图谱和声誉系统

## 六、与主方向 Privacy 的结合

治理 AI 的隐私考虑：

- **Proposal 摘要：** 不暴露投票者的个人立场（匿名投票）
- **Meeting Action：** 转录内容脱敏处理（不记录非公开个人信息）
- **Budget Checklist：** 收款地址验证但不公开关联身份
- **贡献图谱：** 匿名贡献也能积累声誉（Semaphore 等匿名协议）

**核心洞察：** 治理需要透明，但参与者也需要隐私。AI 辅助治理的关键是在信息质量和参与者隐私之间找到平衡。
