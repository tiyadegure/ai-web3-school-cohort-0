## 一、产品背景

**Cobo** 是全球数字资产托管和钱包基础设施领导者，2017 年创立，零安全事故记录，托管数十亿美元资产。

**Cobo Agentic Wallet (CAW)** 于 2026 年 4 月 20 日正式发布——全球首个专为 AI Agent 设计的 MPC 钱包。

---

## 二、三大行业首创

### 1. MPC 非托管钱包

**问题：** 现有 Agent 钱包依赖 TEE、API Key 或委托 EOA——软件级保障，一旦 Agent 被攻破、LLM 被 prompt injection、凭证被盗，资金立即面临风险。

**方案：** CAW 基于 MPC（多方计算）：
- 签名私钥加密分片
- 交易签名由 用户 + Agent + Cobo 基础设施 三方共同参与
- 即使 Agent 被攻破、LLM 幻觉、凭证泄露，也无法单独生成有效签名

**本质区别：** 密码学保证的数学确定性 vs 安全最佳实践

### 2. Pact 协议（人机授权协议）

**问题：** 静态策略系统（白名单/限额）无法适应 Agent 的动态任务需求。

**方案：** 每个任务以 Pact 开始：
- 定义 Agent 能做什么
- 指定执行边界和终止条件
- 按任务动态生成
- 在基础设施层强制执行
- 完成后自动过期

**范式转变：** Agent 从"受限工具"→"在明确定义范围内自主执行的执行者"

### 3. Recipe 驱动的技能层

**问题：** LLM 在链上操作中会幻觉合约地址、伪造 ABI 参数、猜测 Gas 费。

**方案：** Recipe = 模块化"剧本"，预验证的执行路径：
- Aave V3 借贷
- Uniswap V3 / Jupiter swap
- DCA 和网格策略
- Polymarket 预测市场
- x402 和 Stripe 微支付
- Discord 打赏机器人
- 多审批企业工资发放

**升级：** 从"能执行"→"每次都能正确执行"

## 三、两种运行模式

| 模式 | 状态 | 适用场景 | 特点 |
|------|------|----------|------|
| MPC 模式 | 已上线 | 高价值/高风险 | 需用户共同签名，防篡改审计日志，一键撤销 Pact |
| 托管模式 | 即将推出 | 高频/低延迟 | 微支付、DCA、自动打赏 |

## 七、产品研究视角

- **目标用户：** 需要让 AI Agent 处理真实资金的开发者和机构
- **核心价值：** 不是"给 Agent 钱包"，而是"给 Agent 可控的自主权"
- **竞争壁垒：** MPC 基础设施 + Pact 协议 + Recipe 生态的三层护城河
- **潜在风险：** MPC 延迟（多方签名需要通信）可能不适合高频交易场景
