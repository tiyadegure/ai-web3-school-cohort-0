### Cobo Agentic Wallet (CAW) 核心特性

**背景：** Cobo 是全球数字资产托管和钱包基础设施领导者，2017 年创立，零安全事故记录。2026 年 4 月 20 日正式发布 Cobo Agentic Wallet (CAW)——全球首个专为 AI Agent 设计的 MPC 钱包。

#### 三大行业首创

**① 全球首个基于 MPC 的非托管 Agent 钱包**

- 现有 Agent 钱包大多依赖 TEE、API Key 或委托 EOA——这些本质上都是软件级保障
- 如果 Agent 被攻破、LLM 被 prompt injection、凭证被盗，资金立即面临风险
- CAW 基于 MPC（多方计算）：签名私钥被加密分片，交易签名由用户 + Agent + Cobo 基础设施三方共同参与
- 即使 Agent 被攻破、LLM 产生幻觉、凭证泄露，也无法单独生成有效签名
- 这是**密码学保证的数学确定性**，而非安全最佳实践

**② 全球首个 Pact 协议（人机授权协议）**

- 每个任务以 Pact 开始——不只定义 Agent 能做什么，还指定执行边界和终止条件
- 与静态策略系统不同：Pact 按任务动态生成、在基础设施层强制执行、完成后自动过期
- 将 Agent 从"受限工具"转变为"在明确定义范围内自主执行的执行者"
- 新范式：Agent 可以独立行动，同时不牺牲控制权

**③ 全球首个 Recipe 驱动的技能层**

- Recipe = 模块化的"剧本"，为 Agent 钱包场景提供专业执行能力
- 开 Recipe 库覆盖高频场景：
  - Aave V3 借贷
  - Uniswap V3 / Jupiter swap
  - DCA 和网格策略
  - Polymarket 预测市场
  - x402 和 Stripe 微支付
  - Discord 打赏机器人
  - 多审批企业工资发放
- 装备 Recipe 的 Agent 不会幻觉合约地址、伪造 ABI 参数、猜测 Gas 费
- 沿着**验证过的预定义路径**执行——从"能执行"升级到"每次都能正确执行"

#### 两种运行模式

- **MPC 模式（已上线）：** 高价值/高风险操作，交易需要用户共同签名，防篡改审计日志，用户可一键撤销所有活跃 Pact
- **托管模式（即将推出）：** 高频/低延迟场景，适合微支付、DCA、自动打赏

#### 技术集成

- AI 框架：LangChain、OpenAI Agents SDK、Claude MCP、Agno、CrewAI
- 链支持：80+ 主流链（Ethereum、Base、Arbitrum、Optimism、Polygon、Solana）
- Token 支持：3000+

## 💡 我的收获与思考

### 核心洞察

1. **MPC vs TEE 的选择**：CAW 选择 MPC 而非 TEE，因为 MPC 提供密码学保证（数学确定性），而 TEE 是硬件保证（可能有侧信道攻击）。对于处理真实资金的 Agent 钱包，MPC 更安全。

2. **Pact = 可编程的权限边界**：这与 Handbook 中 Agent Wallet 章节的 Session Key 概念一致，但 Pact 更进一步——按任务动态生成、基础设施层强制执行、自动过期。比静态的白名单/限额更灵活。

3. **Recipe = 防幻觉执行**：这是解决 LLM 在链上操作中"幻觉"问题的实用方案。不是让 LLM 更聪明，而是给它预验证的执行路径。从"能执行"到"每次都能正确执行"的跃迁。

4. **与 Week 2 学习的关联**：
   - Agent Wallet 章节：CAW 是 Session Key + MPC 的产品化实现
   - Machine Payment 章节：CAW 支持 x402 微支付，是 Machine Payment 的基础设施
   - Settlement & Escrow 章节：Pact 的"执行边界+终止条件"与 Escrow 状态机的设计理念一致
   - 主方向 Privacy：MPC 的密钥分片天然提供隐私保护

### 产品研究视角

- **目标用户**：需要让 AI Agent 处理真实资金的开发者和机构
- **核心价值**：不是"给 Agent 钱包"，而是"给 Agent 可控的自主权"
- **竞争壁垒**：MPC 基础设施 + Pact 协议 + Recipe 生态的三层护城河
- **潜在风险**：MPC 的延迟（多方签名需要通信）可能不适合高频交易场景
