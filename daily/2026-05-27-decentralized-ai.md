## 核心定义

去中心化 AI 不是"把模型放到链上"，而是在重新设计 AI 系统里数据、模型、算力、推理、评测和收益分配的组织方式。

**一句话：** AI 系统真正需要去中心化的，不一定是模型本身，而是关键资源和关键决策的控制权。

---

## 第一性原理

- 先分层，再判断价值：数据、模型、算力、推理、评测、结算、治理不能混成一个词
- 开放性需要工程代价：分布式网络引入延迟、质量波动、女巫攻击、节点作恶、治理低效
- 链上更适合做协调层：记录权属、激励贡献、结算任务、治理参数，比直接承载大规模模型训练更现实

## 知识节点

### ⭐⭐ Model Market

让模型能力像服务一样被发现、比较、调用和付费的市场。

**关键问题：** 可替换性——把模型选择从硬编码变成路由决策

**路由策略：** 便宜模型负责分类抽取，高能力模型负责复杂推理，本地模型处理隐私内容，专用模型处理代码/图像/链上数据

**价值所在：** 解决发现、评测、路由和结算，不只是模型列表

### ⭐⭐⭐ Data Market

数据如何被授权、定价、验证和复用。

**数据不是普通商品：** 可以被复制、质量难以提前判断、授权边界复杂、隐私和版权风险高

**更现实的设计对象：**
- 可验证来源的数据集
- 带授权条款的训练/微调数据
- 可复现的数据处理 pipeline
- 可审计的数据贡献记录
- 面向特定任务的标注/偏好/评测样本

**链上适合：** 保存授权、哈希、版本、贡献和收益分配，不直接保存原始数据

### ⭐⭐⭐ Compute Market

把 GPU/CPU/存储/带宽/推理服务变成可购买资源的市场。

**难点：**
- 节点性能不稳定，延迟和吞吐会波动
- GPU 型号/驱动/镜像/网络影响可复现性
- 任务结果需要验证，便宜算力可能带来错误输出
- 隐私任务中远程节点不能天然可信
- 生产系统需要 SLA，开放网络难保证

**适合先做的：** 容器化推理、批处理任务、非关键 workloads、可重复验证的任务

### ⭐⭐ Inference Network

把模型推理拆成网络服务：请求→节点执行→返回结果→计费/审计/证明。

**与传统 API 的差别：** 推理节点不一定属于同一家公司，需要处理节点发现、模型版本、请求路由、失败重试、结果验证、付款结算

**关键检查项：**
- 请求发给了哪个模型和节点
- 模型版本是否可追踪
- 失败时能否重试或切换节点
- 隐私数据是否暴露给不可信节点
- 成本/延迟/质量是否被记录

### ⭐⭐⭐ Model Routing

在多个模型、节点或服务之间做选择——去中心化 AI 里容易被低估的一层。

**路由策略考虑维度：**
- 任务类型（总结/翻译/代码/链上分析）
- 风险等级（普通解释 vs 交易决策）
- 数据敏感度（隐私数据→本地模型/TEE）
- 成本预算（低价值任务不调最贵模型）
- 延迟要求（交互式 vs 离线批处理）
- 评测反馈（历史成功率/用户反馈/错误类型）

**本质：** 路由层是 AI 应用的调度器——决定什么时候追求便宜、什么时候追求可靠、什么时候必须拒绝执行

### ⭐⭐ Quality Benchmark

回答：这个模型、节点或数据源到底是否可靠。

**比普通 benchmark 多看的维度：**
- 结果是否稳定
- 节点是否按约定执行
- 延迟和失败率是否可接受
- 输出能否被复现或抽样验证
- 是否存在刷分/过拟合/合谋
- 评测任务是否贴近真实产品场景

**关键原则：** Benchmark 不应只服务排行榜，还应进入路由、定价、声誉和结算

### ⭐⭐ Settlement

从"开放调用"走向"可持续网络"的关键。

**AI 任务结算比普通 API 更复杂：** 结果质量不是二元的（成功但质量差、超时、格式错误、下游验证不可用）

**结算设计包含：** 任务报价和有效期→请求参数和模型版本→输出哈希→状态机（成功/失败/超时/争议）→付款/托管/退款/仲裁规则→任务日志和评测证据

## 判断去中心化 AI 项目的固定问题

1. 它去中心化的是：数据/模型/算力/推理/评测/结算/治理？
2. 链上部分负责：记录/支付/激励/身份/治理/验证？
3. 相比中心化服务，具体改善了：成本/可用性/抗审查/可验证性/贡献分配？
4. 开放网络带来的质量波动和治理成本是否值得？

## 与 Bridge 知识链的关系

```
Chain-aware Context → Web3 Tool Use → Agent Workflow
→ Agent Wallet → Machine Payment → Settlement & Escrow
→ Agent Identity → Agent Trust & Reputation
→ AI Oracle → Verifiable AI → AI Security → AI Privacy
→ AI Sovereignty → Governance AI → Decentralized AI
                                              ↓
                                  最底层基础设施议题
```

**核心洞察：** 去中心化 AI 不一定直接变成用户产品，但会影响 Agent、支付、数据、模型、隐私、验证和治理的基础设施。

## 一句话总结

去中心化 AI = Model Market（模型市场）+ Data Market（数据市场）+ Compute Market（算力市场）+ Inference Network（推理网络）+ Model Routing（模型路由）+ Quality Benchmark（质量评测）+ Settlement（结算）。链上做协调层，不直接承载大模型训练。
