## Handbook Reading

- **Chapter**: [Web3 Tool Use](https://aiweb3.school/zh/handbook/bridge/web3-tool-use/)
- **Key Takeaways**:

### 一句话定义
把 RPC、合约读取、交易生成、钱包确认、区块浏览器和 DeFi 操作变成 Agent 可调用工具的过程。真正难的不是"能调用"，而是权限、参数、模拟和日志。

### 第一性原理
**模型可以选择工具，但工具必须用确定性边界限制模型。**

核心原则：
- 不要让 Agent 直接拼接任意 calldata 或调用任意地址
- 读写分离：读取和发送必须是不同工具、不同权限
- 参数结构化：chain id、contract address、method、args、value、slippage 不能埋在自然语言里
- 日志不可省：每次工具调用都要记录输入、输出、时间、来源和错误

### 8 个知识节点

**RPC Tool**（初级）
读取链状态、查询区块、估算 gas、获取日志或广播交易。只读可以开放，写入必须拆出去。

**Contract Read**（初级）
调用 view/pure 函数，不改变链上状态。读余额、owner、allowance、pool 状态等。风险：读错网络/合约/ABI 不匹配/RPC 滞后。

**Contract Write**（高级）
改变链上状态，必须经严格模拟、权限和确认。写前至少需要：chain id、ABI、value、gas、simulation、policy、授权、回执追踪。Agent 不应有"任意合约写入"能力。

**Wallet Tool**（高级）
最敏感的边界。必须把"连接""签名消息""发送交易""授权 token""撤销授权"分成不同动作。

**Explorer Tool**（初级）
查询交易、合约源码、event、token transfer。提供可验证证据。

**DeFi Tool**（高级）
swap、借贷、授权、仓位查询、清算风险。必须要求：协议白名单、最大交易额、最大滑点、价格来源、simulation、allowance 检查、人工确认。

**Tool Permission**（高级）
定义 Agent 能调用哪些工具、在什么条件下。按工具/合约/方法/金额/时间/频率/确认级别分层。

**Tool Log**（中级）
Agent 行为可审计的基础。每次调用记录：用户目标、工具名、输入、输出、错误、时间、链 ID、区块号、tx hash、确认人、policy 判断。

### 与 Chain-aware Context 的关系
Chain-aware Context 是输入层（Agent 看见什么），Web3 Tool Use 是执行层（Agent 能做什么）。越靠近执行，越需要 policy、simulation、human check 和日志。
