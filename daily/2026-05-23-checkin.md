# 5.23 打卡 | AI × Web3 School Week 1 Day 3

今天最大的收获是：从"Agent 能调用链上工具"到"Agent 需要把操作拆成可控流程"的认知升级。

Handbook Agent Workflow 章节的核心一句话：把概率模型放进确定性流程里。链上操作涉及真金白银和不可逆结果，Agent 不能像聊天一样自由发挥 —— 必须有 Task Graph（步骤拆分）、State Machine（状态追踪）、Human-in-the-loop（风险分层确认）和 Trace（执行日志）。

同天晚上参加了 Open Agentic Economy 活动，Sophia 从 EF 视角讲了 ERC-8004 Agent Identity —— 这正好是 Agent Workflow 在链上的信任基础设施：有了身份和声誉，Workflow 的 trace 才有可验证的归属。

做了什么小行动：画了一张 Agent 辅助 swap 的 8 步流程图，标出了 AI 自动（5步）、人工确认（1步）、钱包签名（1步）、链上确认（1步）的边界。发现 Human-in-the-loop 的关键不是"有没有确认按钮"，而是用户确认时能否看懂资产变化和失败风险。
